2 相关理论与技术
2.1 Java
Java 是一种面向对象的编程语言，具有跨平台、分布式、健壮性等特点。得益于 Java 虚拟机（JVM）的“一次编写，到处运行”机制，它为家政服务平台提供了稳定的后端运行环境。作为一种强类型语言，Java 拥有庞大的生态系统和丰富的类库，特别是在企业级应用开发中，其优秀的内存管理机制和异常处理机制，能够有效保障“贝贝”家政服务平台在高并发场景下的稳定性与可维护性，本项目采用Java 21版本。
2.2 Spring框架
Spring 框架是 Java 平台最为流行的开源应用框架之一，由 Rod Johnson 于 2003 年首次发布。它的核心设计理念是简化企业级应用程序的开发复杂性。Spring 是一个轻量级的控制反转（IoC）和面向切面（AOP）的容器框架，它通过非侵入式的编程模型，使得开发者能够专注于业务逻辑的实现，而无需过多关注底层的资源管理和事务处理。
在“贝贝”家政服务平台中，Spring 框架充当了整个后端架构的“粘合剂”和“基石”。它通过依赖注入（Dependency Injection, DI）机制，实现了组件之间的低耦合，极大地提升了系统的可测试性和扩展性。此外，Spring 庞大的生态系统（如 Spring MVC、Spring Boot、Spring Security 等）为本项目提供了全方位的技术支持。
2.2.1 Spring MVC
Spring MVC 是 Spring 框架中用于构建 Web 应用程序的模块，它遵循模型-视图-控制器（Model-View-Controller, MVC）的设计模式。在本项目中，Spring MVC 负责接收前端 Vue 页面发送的 HTTP 请求，并将其分发给相应的处理器（Controller）。它有效地实现了业务逻辑与数据表现层的解耦，使得代码结构更加清晰。
MVC 模式的核心在于将应用分为三个部分：模型（Model）负责数据处理，视图（View）负责数据显示，控制器（Controller）负责协调工作。

2.2.2 Spring Boot
内容Spring Boot 是由 Pivotal 团队提供的全新框架，其设计目的是用来简化新 Spring 应用的初始搭建以及开发过程。它通过“约定优于配置”（Convention over Configuration）的理念，去除了传统 Spring 开发中繁琐的 XML 配置。 在“贝贝”家政平台的开发中，Spring Boot 的以下特性发挥了关键作用：
自动配置（Auto-configuration）： 能够根据类路径下的 jar 包自动配置 Spring Bean。
内嵌容器： 内置了 Tomcat 容器，无需部署 WAR 文件，直接运行 JAR 包即可启动服务，极大地提升了开发和部署效率。
依赖管理（Starters）： 通过 spring-boot-starter-web 等起步依赖，一站式引入所需组件，避免了版本冲突问题。
2.2.3 Spring Security
Spring Security 是一个功能强大且高度可定制的身份验证和访问控制框架。对于家政服务平台而言，区分“普通用户”、“家政人员”和“管理员”的权限至关重要。 本项目利用 Spring Security 的过滤器链（Filter Chain）机制，实现了基于角色的访问控制（RBAC）。它不仅提供了登录认证功能，还集成了 CSRF（跨站请求伪造）防护和会话管理，确保了平台用户数据的安全性。

2.3 MyBatis
MyBatis 是一款优秀的持久层框架，它支持定制化 SQL、存储过程以及高级映射。与传统的 Hibernate 全自动 ORM 框架不同，MyBatis 允许开发者直接编写 SQL 语句，从而能够对 SQL 进行极致的性能优化，这对涉及复杂查询（如家政订单筛选、地理位置匹配）的场景尤为重要。通过 XML 映射文件或注解，MyBatis 将 Java 对象（POJO）与数据库中的记录进行映射，简化了 JDBC 的繁琐操作。

2.4 MySQL
MySQL 是一种关系型数据库管理系统，以其开源、免费、体积小、速度快等特点，成为 Web 应用的首选数据库。本项目使用 MySQL 存储用户信息、家政服务项目、订单记录等核心数据。MySQL 的 InnoDB 存储引擎支持 ACID 事务特性，能够确保家政订单在生成、支付、完成等状态流转过程中的数据一致性，防止出现数据丢失或错乱。
2.5 Redis
Redis（Remote Dictionary Server）是一个开源的、基于内存的数据结构存储系统，常被用作数据库、缓存和消息中间件。 在“贝贝”家政服务平台中，Redis 主要承担以下角色：
缓存热点数据： 将频繁查询的家政服务类别、首页推荐信息存储在 Redis 中，减少 MySQL 的访问压力，降低系统响应延迟。
令牌存储： 设置带有过期时间的 Key 来存储登录 Token，实现高效的会话验证。

2.6 JWT
JWT 是一种用于在网络应用环境间传递声明的基于 JSON 的开放标准（RFC 7519）。在前后端分离的架构中，传统的 Session 方式因为状态存储在服务器端，不利于水平扩展。 JWT 采用无状态的认证机制，服务端在认证通过后生成一个包含用户信息的加密 Token 返回给前端。前端后续请求都在 Header 中携带该 Token。JWT 由头部（Header）、载荷（Payload）和签名（Signature）三部分组成。
2.7 Vue
Vue.js 是一套用于构建用户界面的渐进式 JavaScript 框架。与传统直接操作 DOM 的开发方式不同，Vue 采用了 MVVM（Model-View-ViewModel）模式，通过双向数据绑定，使得视图层（View）和数据层（Model）能够自动同步。 本项目利用 Vue 的组件化开发特性，将页面拆分为导航栏、服务卡片、订单列表等独立组件，极大地提高了前端代码的复用性和可维护性。同时，配合 Axios 库与后端进行异步数据交互，为用户提供了流畅的单页应用（SPA）体验。
2.8 本章小结
本章主要阐述了“贝贝”家政服务平台设计与实现所依托的关键技术栈。系统后端采用 Java 语言，利用 Spring Boot 框架整合 Spring MVC、Spring Security 和 MyBatis，构建了高效稳定的服务端架构，并结合 MySQL 与 Redis 确保数据持久化与高性能访问；前端则基于 Vue.js 框架实现了组件化开发与前后端分离交互。这些成熟技术的综合应用，有效解决了系统高并发、安全性及扩展性问题，为后续的功能开发与系统实现奠定了坚实的技术基础。
